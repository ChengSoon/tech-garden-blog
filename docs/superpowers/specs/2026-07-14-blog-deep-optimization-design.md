# 技术刊深度体验优化 — 设计说明

**日期：** 2026-07-14

**状态：** 已批准，待实施计划

**方案：** 编辑刊物 × 知识地图
**基线：** 延续 `2026-07-13-blog-design.md` 的内容模型与静态架构，重构其界面、交互与功能体验

## 1. 目标与边界

### 1.1 核心目标

站点按以下优先级优化：

1. **读：** 让访客更快找到值得读的内容，并获得稳定、沉浸的长文体验。
2. **逛：** 用主题与显式链接把一次阅读延伸为连续探索。
3. **识：** 通过内容气质、Now 与唱机建立作者记忆，而非堆砌个人宣传。

### 1.2 产品定位

> 一本具有独立编辑秩序的技术刊物，也是一张可以沿关联关系漫游的知识地图。

### 1.3 设计约束

- 保持 Astro 静态生成，不新增账号、数据库、评论、复杂 CMS 或运行时 API。
- 保留首页唱机的辨识度，但降低默认动态；完整体验集中到 `/now`。
- 现有 Markdown/MDX、Content Collections、系列、标签、RSS 与 Cloudflare 部署流程继续有效。
- 核心内容在 JavaScript 关闭时仍可访问；交互能力以渐进增强方式加入。
- 不引入重型图谱物理引擎，不以动画、装饰或卡片数量掩盖内容不足。

### 1.4 不在本轮范围

- 邮件新闻信后端、用户登录、跨设备同步、评论与浏览量。
- 内容管理后台、数据库迁移、内容模型大改。
- 3D 图谱、持续物理模拟、滚动劫持或自动播放音频。
- 大规模重写现有文章正文。

## 2. 当前问题

- 首页的刊物 Hero、全屏唱机、精选卡片与花园预览同时争夺注意力，主路径不够明确。
- 多数区段使用相似卡片、胶囊按钮与悬浮反馈，层级主要依赖容器而非编辑版式。
- H1 描述价值主张但没有直接建立 “Cheng · 技术刊” 品牌。
- `/articles` 主要是年份归档；搜索仅覆盖标题、摘要与标签，内容发现能力有限。
- 花园目前以时间线和书架为主，关联关系没有成为可感知的探索入口。
- 阅读字号、花园视图等本地状态分散管理，缺少收藏、历史与进度闭环。
- 唱机、花园与部分 CSS 文件超过项目文件长度约束，继续叠加功能会增加维护风险。

## 3. 体验原则

1. **一个区段，一个动作：** 每个首页区段只负责开始阅读、查看更新、探索关联、继续系列或订阅。
2. **内容先于容器：** 优先使用标题、编号、网格、细线与留白；只有独立实体使用卡片。
3. **动效由用户触发：** 唱机播放后才进入完整运动，其余动效用于状态变化和层级过渡。
4. **关系必须可解释：** 推荐与邻居来自标签、系列或显式链接，不使用黑盒推荐。
5. **本地增强可丢失：** 收藏、历史和偏好提升体验，但不可成为访问内容的前提。

## 4. 信息架构

### 4.1 顶部导航

主导航保持五个入口：

- **阅读** `/articles`：精选、最新、筛选、稍后读。
- **花园** `/garden`：主题地图与生长时间线。
- **系列** `/series`：有序专题与继续阅读。
- **Now** `/now`：当前研究、近期更新与完整唱机。
- **关于** `/about`：作者身份、方向与代表作。

搜索、主题切换和移动菜单使用图标按钮；按钮具备可访问名称、悬停提示和可见焦点。

### 4.2 首页叙事

首页固定为以下顺序：

1. **品牌首屏：** “Cheng · 技术刊”、一句价值主张、最新主打文章和静态唱机信号。
2. **继续阅读：** 仅在本地存在未读完记录时出现。
3. **从这里开始：** 三条人工策展阅读路径。
4. **最新生长：** 按时间混排 Essay 与 Note，使用可扫描的编辑列表。
5. **知识地图：** 展示真实主题簇和代表节点，进入完整花园。
6. **系列书架：** 展示章节进度和下一篇入口。
7. **Now / Listening / RSS：** 用紧凑编辑区完成作者记忆和回访转化。

首屏不是卡片，最大高度约为桌面 `76svh`、移动端 `70svh`；两个尺寸都必须露出下一内容区的标题或边缘。

## 5. 视觉系统

### 5.1 美学方向

主轴为 **冷静编辑部**，辅以 **模拟仪器细节**。页面像独立技术刊而非 SaaS 模板；唱片纹理、编号和等宽标注是点缀，不形成整站复古主题。

### 5.2 色彩

- 浅色基底：冷灰纸白；正文使用近黑墨色。
- 深色基底：炭黑；正文使用柔白。
- 主强调：深松绿，用于链接、当前状态和关系节点。
- 信号色：克制朱红，仅用于播放、重点进度和需要注意的动作。
- 生长状态继续使用三种低饱和状态色，但不承担正文语义。
- 移除大面积米黄、渐变光斑和独立装饰色球。

### 5.3 字体与网格

- 展示字保持有编辑气质的衬线字体；中文标题与正文使用可读性稳定的中文衬线回退。
- UI 使用清晰无衬线，代码与技术标注使用等宽字体。
- 桌面采用 12 列、移动端 4 列；页面宽度约 `1200px`，正文保持 `66–70ch`。
- 圆角控制在 `0–6px`；标签和状态可使用胶囊，其余按钮与容器不默认胶囊化。

### 5.4 动效与签名时刻

- 首屏只执行一次有顺序的标题、主打文章与唱机信号进入动画。
- 首页唱机默认静止；播放后唱片旋转、唱臂落下并进入完整状态。
- `/now` 保留沉浸唱机，但正文、控制区和导航始终清晰可用。
- 列表悬停优先改变线条、颜色或指示符，不普遍使用卡片上浮。
- `prefers-reduced-motion` 下关闭旋转、位移和弹簧动效。

## 6. 页面设计

### 6.1 阅读中心 `/articles`

- 顶部提供精选、最新、长文、笔记和稍后读入口。
- 筛选维度为类型、主题、状态和年份；筛选结果与 URL 查询参数同步。
- 默认使用紧凑列表，展示标题、摘要、日期、类型、状态和阅读进度。
- 没有匹配结果时保留清除筛选动作，并给出其他可浏览主题。

### 6.2 文章页

- 文首层级为类型、标题、摘要、日期、阅读时长、状态和标签。
- 桌面右侧保留目录；移动端使用可展开目录，不遮挡正文。
- 阅读工具包含收藏、字号、栏宽、复制链接和原生分享。
- 滚动进度按 slug 保存，回访时提供“从上次位置继续”，不自动跳转。
- 文末顺序为系列导航、显式邻居、反向链接、作者与订阅入口。

### 6.3 花园 `/garden`

- 两种视图为“主题地图”和“生长时间线”，记忆用户上次选择。
- 主题地图使用构建期确定布局的轻量 SVG；不运行持续物理模拟。
- 有有效保存偏好时优先恢复；没有偏好时桌面默认地图，窄屏和减弱动效环境默认时间线。
- 类型、状态、标签与搜索共享同一筛选状态；选中节点时突出一跳邻居。
- 地图无法初始化时直接显示时间线，不阻断页面。

### 6.4 Now、关于与订阅

- `/now` 包含当前研究主题、最近更新、正在阅读/聆听和完整唱机。
- `/about` 采用编辑档案形式，突出一句身份、研究方向和三篇代表作。
- `/subscribe` 只承诺 RSS：复制源地址，并提供 Feedly、Inoreader 等快捷入口。
- 页面不展示解释视觉元素或界面操作的冗长说明文案。

## 7. 功能设计

### 7.1 本地阅读状态

使用单一版本化存储键管理以下数据：

```ts
interface ReaderStateV1 {
  version: 1;
  savedSlugs: string[];
  history: Array<{
    slug: string;
    visitedAt: string;
    progress: number;
  }>;
  preferences: {
    fontScale: number;
    lineWidth: 'narrow' | 'normal' | 'wide';
    gardenView: 'map' | 'timeline';
  };
}
```

- `progress` 限制在 `0–1`；历史按最近访问排序，最多保留 50 条。
- 所有读写经过解析、校验和异常捕获；无效版本回退为默认值。
- 状态变化通过统一浏览器事件通知首页、归档和文章工具更新。

### 7.2 搜索

- 构建期生成静态搜索索引，包含标题、摘要、标签、类型和去标记正文。
- 搜索弹窗首次打开时按需加载索引，避免在每页 HTML 中内嵌完整正文数据。
- 结果按标题、标签、摘要、正文的命中权重排序，并按 Essay / Note 分组。
- 支持方向键、Enter、Escape、焦点锁定与关闭后焦点恢复。
- 索引加载失败时退回现有元数据搜索，并提供进入 `/articles` 的入口。

### 7.3 策展与关联

- “从这里开始”由明确的数据配置维护，每条路径包含标题、说明和有序 slugs。
- 邻居排序依次考虑显式链接、同系列和共同标签，并在界面标明关联原因。
- 空路径、失效 slug 或草稿内容在构建时过滤，不生成失效链接。

## 8. 技术结构与数据流

```text
Content Collections
  ├─ published posts ──► 页面、RSS、系列、标签
  ├─ body + metadata ──► 静态搜索索引
  ├─ links + tags ─────► 图谱、主题簇、邻居原因
  └─ curated paths ────► 首页“从这里开始”

浏览器
  ├─ localStorage ─────► 收藏、历史、进度、阅读偏好
  └─ React islands ────► 搜索、花园、唱机
```

- Astro 负责页面与静态数据生成；React 不接管整页路由。
- 主题继续使用独立的 `theme` 键在页面首屏前同步应用，避免与阅读状态合并后产生主题闪烁。
- 首页唱机使用静态首帧并延后 hydrate；`/now` 可立即加载交互。
- 花园岛屿进入视口后加载；搜索索引仅在弹窗首次打开时请求。
- 主题、收藏和阅读偏好不改变 URL；归档筛选使用 URL 查询参数，便于分享和返回。

## 9. 异常与降级

- localStorage 不可用：隐藏继续阅读与收藏状态，阅读和导航保持正常。
- 搜索索引失败：退回元数据搜索；无 JavaScript 时从阅读中心浏览。
- 关系图失败或不适配：展示时间线和文本邻居。
- 媒体加载失败：唱机展示封面、曲名与可访问的失败状态，不阻塞其他页面内容。
- 剪贴板失败：选中可复制文本并保留原始链接。
- 筛选为空：显示清除筛选和可替代主题，不显示死端空白页。

## 10. 可访问性与性能

- 所有图标按钮具备可访问名称、键盘状态、可见焦点和必要提示。
- 弹窗实现焦点锁定、Escape 关闭、背景不可交互及关闭后焦点恢复。
- 移动端交互目标至少 `44px`；正文和控件满足合理对比度。
- 不自动播放音频，不以颜色作为唯一状态信号。
- 避免固定大面积模糊、持续背景重绘和未触发的无限动画。
- 拆分超过 300 行的相关组件与样式文件；新函数控制在 50 行以内。

## 11. 验证与验收

### 11.1 自动验证

- 为阅读状态解析、进度裁剪、历史排序、搜索排序和关联原因补充单元测试。
- 保持现有链接、图谱、布局和阅读时长测试通过。
- 执行 TypeScript 检查、完整测试和 Astro 生产构建。

### 11.2 视觉与交互验证

- 检查桌面、平板和手机尺寸下的首页、归档、文章、花园、Now 与搜索。
- 检查浅色、深色、减弱动效、键盘导航和焦点顺序。
- 检查空内容、无历史、无收藏、无搜索结果、存储失败和媒体失败。
- 确认所有页面无水平溢出，首屏能露出下一内容区，正文不被固定控件遮挡。

### 11.3 完成标准

1. 首页形成“品牌 → 阅读 → 更新 → 探索 → 系列 → 回访”的单向清晰叙事。
2. 唱机保留辨识度，但未播放时不与内容争夺注意力。
3. 访客能搜索、筛选、收藏、恢复阅读并从关联原因继续探索。
4. 核心页面在无 JavaScript、窄屏和减弱动效环境下仍可用。
5. 相关测试、类型检查、生产构建与响应式视觉检查均有真实证据。
