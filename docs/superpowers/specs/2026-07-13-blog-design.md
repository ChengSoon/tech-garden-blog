# 个人技术博客 + 数字花园 — 设计说明

**日期：** 2026-07-13  
**状态：** 待用户确认（brainstorm 定稿）  
**范围：** 绿场个人站点；技术博客为主，数字花园为辅  

---

## 1. 背景与目标

### 1.1 问题

需要一个有设计感、且具备辨识度功能的个人博客站点。常见模板站「列表 + 文章页」无法同时满足：

- 长文阅读的杂志级体验
- 个人知识沉淀与互链探索（数字花园）
- 适度的「有意思」交互，又不牺牲内容与可维护性

### 1.2 定位（一句话）

> 用内容驱动的静态优先架构，做一本**可漫游的技术刊**：杂志级阅读为骨，双向链接与花园星座为魂。

### 1.3 受众与成功标准

| 角色 | 诉求 |
|------|------|
| 访客（开发者为主） | 快速判断「这人写什么」、沉浸读完一篇、能顺着主题逛下去 |
| 作者（你） | Markdown 写作、构建即发布、结构清晰、少运维 |

**成功标准：**

1. 首屏有「刊」的质感，而非通用博客模板
2. 任意内容页都能进入花园邻域（邻居节点 / 主题）
3. 内容较少（少于 15 篇）时仍不空洞（精选 + 主题桩）
4. 有趣功能服务「读」与「逛」，不抢正文
5. 第一期可独立部署上线，无后端强依赖

### 1.4 明确不做（全局）

- 多作者 / 复杂 CMS / 账号体系
- 重社交（关注、动态流）
- 第一期评论、新闻信、浏览量统计（可二期）
- 3D 宇宙式图谱、滚动劫持、首屏重动画

## 2. 核心体验与信息架构

### 2.1 站点三角

| 面 | 名称 | 职责 |
|----|------|------|
| 刊 | Blog | 长文、系列、教程 — 深度阅读 |
| 园 | Garden | 节点、主题星座、互链 — 探索与回访 |
| 人 | You | About / Now — 身份与当下焦点 |

### 2.2 三条主路径

| 路径 | 入口 | 行为 | 终点 |
|------|------|------|------|
| 读 | 首页精选 / 文章归档 | 沉浸阅读长文 | 系列下一篇 / 邻居节点 |
| 逛 | 花园（星座或网格） | 从图或标签漫游 | 预览 → 全文 |
| 识 | About / Now | 了解作者与近况 | 代表作或当前研究主题 |

### 2.3 导航

**顶栏：**

```
[Logo/名字]    文章    花园    系列    关于      [搜索] [主题切换]
```

**页脚（极简）：** RSS · GitHub · （可选）邮箱

### 2.4 内容类型

| 类型 | 含义 | 呈现 |
|------|------|------|
| Essay | 完整长文 | 杂志级正文模板 |
| Note | 短笔记 / 半成品 | 轻模板 + 强调反链与状态 |
| Series | 有序 Essay 集合 | 系列封面 + 章节列表 |
| Topic | 主题聚合（逻辑簇） | 星座中心 / 主题页（可由标签或 frontmatter 导出） |

Essay 与 Note 均进入图谱；Essay 侧重排版，Note 侧重关联与生长状态。

**Topic（MVP 简化）：** 不强制独立内容文件。第一期由高频 `tags` 或 frontmatter 中的主题簇生成聚合入口；若某 tag 下 ≥2 篇内容，花园中可显示为簇。独立 Topic 页可作为 P1。

### 2.5 首页结构

1. **Hero**：一句话定位 + 双 CTA（读精选 / 逛花园）
2. **本季精选**：1 篇主打 + 2–3 篇卫星
3. **花园一瞥**：简化星座或「最近生长的节点」
4. **进行中的系列**
5. **Now** 一行（链到完整 Now）

## 3. 关键页面与功能设计

### 3.1 Essay 页（P0）

**桌面布局：**

- 主栏正文宽度约 65–72ch
- 可选左侧旁注（术语/补充）；移动端降为脚注或点击展开
- 右侧章节轨：滚动高亮当前 `h2`；顶部细阅读进度条

**文首 meta：** 日期 · 阅读时长 · 标签 · 所属系列 · 生长状态（seedling / growing / evergreen）

**文末：**

- 系列上一篇 / 下一篇（若有）
- **邻居节点** 3–6 个（来自显式双向链接图，非黑盒推荐）
- 进入所属主题 / 花园的入口

**代码块：** 语言标签、复制按钮、可选行号；保持高对比可读。

### 3.2 Note 页（P0）

- 轻装饰；顶部生长状态明确
- **反向链接列表**（哪些内容链到此节点）
- 同屏一跳邻域小图（径向或简化力导向的静态/轻交互 SVG）

### 3.3 花园页（P0）

**双视图**（持久化用户上次选择到 `localStorage`）：

| 视图 | 形态 | 用途 |
|------|------|------|
| 星座图 | 节点 + 边；主题成簇 | 探索与品牌辨识 |
| 网格 | 卡片 + 状态色点 + 筛选 | 高效检索 |

**交互（MVP）：**

- 悬停：标题、类型、摘要
- 点击：侧滑预览 → 进入全文
- 筛选：类型、标签、生长状态
- 空态 / 冷启动：主题桩 + 虚位说明，避免荒凉

**降级：** 图谱初始化失败或 `prefers-reduced-motion` / 窄屏策略下，可默认网格或简化静态图。

**不做（一期）：** 3D、重物理引擎、登录墙。

### 3.4 系列（P1，但信息架构预留；MVP 建议带上最小版）

- 系列封面：引言 + 进度（已发布 n 篇）
- 有序章节列表
- CTA：从第 1 章读

MVP 若时间紧：可先「系列字段 + 文内前后导航 + 简单系列列表页」，封面视觉可后补。

### 3.5 About / Now（P0）

- **About**：人话介绍 + 代表作 3 篇 + 兴趣/方向标签（避免简历堆砌）
- **Now**：本季在研究什么（手动更新）

### 3.6 归档与搜索

- **文章页**：时间线 + 标签 / 年份筛选（P0）
- **搜索**：标题 + 标签（P0 可接受）；全文搜索（P1，Pagefind / flexsearch）

### 3.7 特色能力优先级

| 优先级 | 能力 | 说明 |
|--------|------|------|
| P0 | 双向链接 + 文末邻居 | 花园成立基础 |
| P0 | 杂志级文章页（TOC / 进度 / 排版） | 技术刊质感 |
| P0 | 花园双视图 | 主辨识度 |
| P0 | About / Now / 归档 / 深浅色 / RSS / 基础 SEO | 站点完整 |
| P1 | 系列完整体验、全文搜索、阅读字号调节 | 体验加深 |
| P2 | 终端彩蛋、文内可运行 demo 槽位 | 趣味点缀 |
| P2 | Git 写作时间线 / 版本感 | 创作痕迹 |

**彩蛋原则：** 默认隐藏；不挡阅读；不进主导航。

### 3.8 作者工作流

1. 在仓库 `src/content/`（或约定目录）撰写 MD/MDX
2. 填写 frontmatter（见 §5）
3. `links` 声明出链 slug → 构建生成 `backlinks` 与 `graph` 数据
4. 本地预览 → Git 推送 → 静态托管部署

## 4. 视觉系统

### 4.1 气质

**克制编辑感为主，极客细节为辅。**  
安静、近纸感、清晰层级；代码与图谱带轻微「仪器感」，不做整站终端皮或赛博喊麦。

### 4.2 色彩

- 浅色：暖纸白背景 + 高对比墨色正文（避免纯黑死板）
- 深色：石墨底 + 柔和前景；代码块表面略抬升
- **单一强调色**（建议靛蓝或松绿）：链接、进度、图谱高亮、focus ring
- 状态色低饱和：seedling / growing / evergreen

### 4.3 字体

- 正文：优质中文回退栈（实现阶段在「现代黑体」与「温和衬线」中选定一套并全局统一）
- 标题：同族或略有性格的无衬线，靠字重与留白而非巨大字号
- 代码 / 图谱标注：等宽（如 JetBrains Mono / IBM Plex Mono）

### 4.4 版式与组件

- 正文栏 65–72ch；段落节奏偏编辑刊物
- 小圆角、极淡阴影；分隔优先留白与细线
- 卡片与导航视觉重量低于正文

### 4.5 动效

- **允许：** 淡入、悬停、侧滑预览、主题切换过渡
- **禁止：** 首屏大动画挡内容、滚动劫持、强视差
- 遵循 `prefers-reduced-motion`

### 4.6 图谱视觉

- 2D SVG/Canvas；Essay 实心节点，Note 空心或更小
- 边默认极低对比，悬停增强当前邻域
- 主题簇淡色晕染，避免标签墙

## 5. 内容模型（Frontmatter）

```yaml
title: string              # 必填
date: date                 # 必填
type: essay | note         # 必填
status: seedling | growing | evergreen  # 必填，默认 growing
tags: string[]             # 建议
summary: string            # 建议；用于卡片、OG、预览
series: string?            # series id
order: number?             # 系列内顺序
links: string[]            # 出链 slugs（显式）
hero: string?              # 可选封面图路径
draft: boolean?            # 默认 false；true 不进生产构建
```

**构建产物（逻辑）：**

- `graph.json`：nodes + edges
- `backlinks` 映射：slug → 链入列表
- 搜索索引（P0 可仅元数据；P1 全文）
- RSS feed

**链接规则：**

- 正文内链与 frontmatter `links` 应可归一（实现计划中约定以何者为图谱边的权威源；推荐：构建时解析 MD 内链 **并** 合并 `links` 字段，去重）

## 6. 技术架构

### 6.1 推荐栈

| 层 | 选择 | 理由 |
|----|------|------|
| 框架 | Astro (TypeScript) | 内容站性能；岛屿架构按需 hydrate |
| 内容 | MD / MDX + Content Collections | 类型化 schema；MDX 预留 demo 槽位 |
| 样式 | Tailwind + 文章专用排版层 | 工程效率 + 长文可读性 |
| 图谱 UI | 构建期数据 + 轻量 SVG/Canvas 岛屿 | 静态数据、运行时轻 |
| 搜索 | Pagefind 或 flexsearch（P1 全文） | 无后端 |
| 部署 | Cloudflare Pages / Vercel / GitHub Pages | 静态托管 |
| 包管理 | pnpm 或 npm（实现时固定一种） | — |

**备选：** Next.js App Router + MDX — 仅当强依赖重型 React 生态时再考虑；默认不采用。

### 6.2 逻辑架构

```
Markdown/MDX ──► Content Collections
                    │
                    ├─► 页面路由（首页/文章/花园/系列/关于）
                    ├─► graph.json + backlinks
                    ├─► RSS
                    └─► search index
                           │
                           ▼
                    静态托管 CDN
                           │
                    客户端岛屿：主题、花园图、搜索 UI、侧滑预览
```

### 6.3 非功能要求

- 文章页以静态 HTML 为主，JS 按需
- 可访问性：键盘可达导航与预览关闭；对比度达标；focus 可见
- SEO：每页 title/description；OG；语义化标题层级
- 性能：避免为整站加载图谱库；花园页再加载图谱岛屿

## 7. MVP 范围与分期

### 7.1 第一期（可发布）

1. 项目脚手架（Astro + TS + Tailwind + Content Collections）
2. 全局布局、导航、页脚、深浅色
3. 首页（§2.5）
4. Essay / Note 双模板（TOC、进度、邻居、系列前后）
5. 文章归档 + 标签
6. 花园页：网格 + 简化星座 + 侧滑预览
7. 系列最小闭环（列表 + 详情或等价导航）
8. About + Now
9. RSS + 基础 SEO
10. 响应式与 reduced-motion 降级
11. 示例内容：≥1 Essay、≥3 Note、≥1 主题簇、写好 About/Now

### 7.2 第二期+

- 全文搜索增强、阅读字号
- 终端彩蛋、MDX 可运行 demo 槽位
- Git 历史时间线
- 评论 / 订阅 / 统计（若仍需要）
- 图谱布局算法与动效增强（仍保持 2D、可关）

### 7.3 冷启动内容策略

上线前手工准备代表作与互链，使花园至少有一个可读的「星座」；首页精选不依赖算法。

## 8. 风险与约束

| 风险 | 缓解 |
|------|------|
| 内容少时图谱空洞 | 主题桩、精选首页、网格默认可选 |
| 图谱库过重 | 自研轻量 SVG 或极小依赖；仅花园页加载 |
| 中文排版与字体 | 系统栈 + 明确回退；正文宽度与行高单独调 |
| 互链维护成本 | 构建校验 dead link；允许仅 frontmatter links |
| 范围膨胀（彩蛋/AI/评论） | 严格按 P0/P1/P2；一期不做 P2 |

## 9. 验收清单（设计层）

- [ ] 陌生人 10 秒内能说出站点主题与「先读哪篇」
- [ ] 一篇 Essay 读完可到达至少 2 个相关节点
- [ ] 花园可切换星座/网格，筛选可用
- [ ] 深浅色下正文与代码对比度可接受
- [ ] 无 JS 或图谱失败时，仍可通过网格/归档读完所有文
- [ ] 作者仅靠改 Markdown + 推送即可更新站点
